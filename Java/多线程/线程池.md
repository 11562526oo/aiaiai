## 1. corePoolSize（核心线程数）

- 线程池中**长期保持存活**的线程数量
- 即使空闲也不会被回收（默认情况下）
- 当任务到来时，优先创建核心线程执行

## 2. maximumPoolSize（最大线程数）

- 线程池允许创建的**最大线程数量**
- 当：
    - 核心线程都在忙
    - 队列已满  
        才会继续创建线程，直到达到最大线程数

## 3. keepAliveTime（线程存活时间）

- **非核心线程**空闲后允许存活的时间
- 超过这个时间会被回收

## 4. unit（时间单位）

- keepAliveTime 的时间单位
- 常见：
    - `TimeUnit.SECONDS`
    - `TimeUnit.MILLISECONDS`
    - `TimeUnit.MINUTES`

## 5. workQueue（任务队列）

- 用于保存等待执行任务的 **阻塞队列**

常见队列：

- `ArrayBlockingQueue`（有界队列）
- `LinkedBlockingQueue`（默认无界）
- `SynchronousQueue`（不存任务，直接交给线程）
- `PriorityBlockingQueue`（优先级队列）

## 6. threadFactory（线程工厂）

- 用来**创建线程**
- 可以自定义线程名字、优先级等

示例：

复制

`new ThreadFactory() {     public Thread newThread(Runnable r) {         return new Thread(r, "my-thread");     } }`

## 7. handler（拒绝策略）

当 **线程数达到最大 + 队列已满** 时触发。

JDK 提供 4 种：

1. `AbortPolicy`（默认）
    
    - 直接抛异常
2. `CallerRunsPolicy`
    
    - 由提交任务的线程自己执行
3. `DiscardPolicy`
    
    - 直接丢弃任务
4. `DiscardOldestPolicy`
    
    - 丢弃队列最旧任务，再提交新任务

## 线程池执行流程（面试高频）

任务提交后执行顺序：

1. **线程数 < corePoolSize** → 创建核心线程执行
2. **线程数 ≥ corePoolSize** → 任务进入队列
3. **队列满了** → 创建新线程（直到 maximumPoolSize）
4. **线程数 = maximumPoolSize 且队列满** → 执行拒绝策略
