ACID：原子性、一致性、隔离性、持久性

事务的隔离级别：
	- 读未提交 
	-             **脏读（Dirty Read）**：读取到未提交的数据，如果对方回滚，数据就  是无效的
	- 读已提交
	-             可能导致不可重复读 和 幻读
	- 可重复读
	-              可能导致幻读
	- 串行化

**主要靠四个组件Redo Log、Undo Log、锁、MVCC**

1. Undo Log  保证原子性。每次修改之前把原来的值存在Undo Log中。事务回滚时，按照Undo Log 反向操作恢复数据，要么全做完，要么全撤销。不会存在中间的状态。
2. Redo Log   保证持久性。将数据先写入Redo Log。就算重写数据宕机，重启也可以通过回放Redo Log就可以恢复数据
3. 锁机制  保证隔离性
4. 一致性是由共同结果


**MVCC多版本并发控制**
	数据库通过保存数据的多个版本，让读写操作不互相阻塞，提高并发性能
