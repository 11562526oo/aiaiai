**问题：**
``` mysql
为什么这两个效率差很多
EXPLAIN SELECT * FROM work_order WHERE code = '6155'; 
EXPLAIN SELECT * FROM work_order WHERE code = 6155;
```

排查思路：
explain慢查询
`system` > `const` > `eq_ref` > `ref` > `range` > `index` > `ALL`
1. **system / const**  
    最多读 1 行（或常量表）。典型：按主键/唯一索引等值查。
2. **eq_ref**  
    关联查询时，被驱动表按主键/唯一索引精确匹配，每次最多 1 行。
3. **ref**  
    用非唯一索引等值查，可能匹配多行（例如 `user_id=?`）。
4. **range**  
    范围扫描（`> < BETWEEN IN(常量列表) LIKE 'abc%'` 等）。
5. **index**  
    全索引扫描（把整棵索引从头到尾扫一遍），比全表扫好一点点（但仍可能很慢）。
6. **ALL**  
    全表扫描（最差的一类，数据大时通常要优化）。

SHOW INDEX查看索引
**Extra**:看会不会回表  Using index   ，using where
